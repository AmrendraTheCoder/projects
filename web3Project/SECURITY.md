# Security Policy

## Reporting a Vulnerability

**Please do not open public issues for security vulnerabilities.**

If you discover a security vulnerability, please email us at:
**security@adaptiveyield.xyz**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will respond within 48 hours and work with you to address the issue.

## Scope

Security vulnerabilities in:
- Smart contracts
- Frontend application
- Deployment scripts
- Access control mechanisms
- Economic exploits

## Bug Bounty

We plan to launch a bug bounty program after our security audit. Stay tuned!

## Security Best Practices

### For Users

1. **Never share private keys** - We will never ask for your private keys
2. **Verify contract addresses** - Always check official sources
3. **Start small** - Test with small amounts first
4. **Use hardware wallets** - For significant amounts
5. **Keep software updated** - Update MetaMask and browsers regularly

### For Developers

1. **Code Review** - All changes must be reviewed
2. **Testing** - Comprehensive test coverage required
3. **Dependencies** - Keep dependencies updated
4. **Access Control** - Minimize privileged functions
5. **Documentation** - Document security considerations

## Known Risks

### Smart Contract Risks

1. **Protocol Risk** - Dependent on underlying DeFi protocols
2. **Smart Contract Bugs** - Despite testing, bugs may exist
3. **Admin Keys** - Owner has privileged access (will use multi-sig)
4. **Gas Costs** - Variable and may be high on mainnet

### Economic Risks

1. **Impermanent Loss** - N/A (stablecoins only)
2. **Slippage** - Large rebalances may experience slippage
3. **Market Risk** - APYs fluctuate with market conditions
4. **Gas Price Risk** - High gas may delay rebalancing

We follow responsible disclosure practices:
1. Acknowledge receipt within 48 hours
2. Provide estimated fix timeline
3. Keep researcher updated on progress
4. Credit researcher (if desired) after fix
5. Coordinate public disclosure
